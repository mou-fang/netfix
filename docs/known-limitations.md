# 已知限制

> 对应任务书 §0.2。记录当前不能检测或不能安全修复的问题。

## 阶段 0 限制

- **无任何诊断能力**：尚未实现任何探针，不能检测网络问题。
- **无任何修复能力**：尚未实现任何修复动作。
- **无历史记录**：尚未实现本地存储。
- **无报告导出**：尚未实现报告与脱敏。
- **无 Windows VM 测试**：无法进行故障注入测试。

## 待验证事项（不得标记为已通过）

以下三项必须保留，直到有实际证据：

- **NetMedic.Core 尚未在真实 Linux 环境运行测试。** `net10.0` 目标框架只保证可编译和可跨平台设计，不能代替实际在 Linux 上运行测试验证。
- **GitHub Actions 尚未实际执行。** `.github/workflows/ci.yml` 只是 ready-to-enable 配置文件，从未被真实 CI 触发过。
- **Windows VM 尚未配置。** 必须在阶段 2（真实 Windows 探针）和阶段 4（修复功能）之前解决，否则无法做故障注入测试和快照恢复。

## 后续阶段将持续更新的固有限制（任务书规定的安全边界）

- 不提供 VPN、代理节点或封锁绕过。
- 不自动卸载 VPN 驱动、虚拟网卡、安全软件。
- 不自动执行 Winsock/TCP-IP/全网络重置（只引导）。
- 不修改企业网络、组策略、静态 IP。
- 不登录或修改路由器后台。
- 不抓包、不做 HTTPS 中间人、不安装根证书。
- 不承诺清除病毒。
