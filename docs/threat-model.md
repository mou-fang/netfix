# 威胁模型（最小化）

> 对应任务书 §9。阶段 0 尚无修复能力，仅记录后续阶段需防护的威胁，为设计预留。

## 威胁清单

| 威胁 | 说明 | 防护阶段 |
|---|---|---|
| 恶意 URL/路径注入 | 用户输入的 URL、域名、应用路径被拼入命令或注册表 | 阶段 2（URL 解析） |
| UAC Worker 伪造 | 普通进程伪造提权请求 | 阶段 4（nonce/过期/白名单） |
| 请求重放/过期 | 捕获提权请求文件后重放 | 阶段 4 |
| 不可信 PAC 执行 | PAC 脚本被当作代码执行 | 阶段 2（只解析不执行） |
| 日志泄露隐私 | 用户名/MAC/SSID/代理凭据写入日志 | 阶段 6（脱敏） |
| 半修改状态 | 修复中崩溃留下不一致配置 | 阶段 4（事务/崩溃恢复） |
| 通用 shell 暴露 | 前端/UI 能调用任意命令 | 不适用（WPF 无前端 shell，阶段 0 已避免） |

## 阶段 0 已实施的防护

- app.manifest `asInvoker`：不默认提权。
- 无 shell 执行、无进程启动、无网络请求。
- TreatWarningsAsErrors：强制代码质量。
